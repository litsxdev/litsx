# LitSX 0.1.0

LitSX 0.1.0 is the first public release wave of the LitSX authored tooling stack.

## What LitSX Is

LitSX is a Lit-first authored JSX surface for web components. It combines:

- Lit-flavoured JSX bindings such as `@event`, `.prop`, and `?attr`
- static hoists such as `^styles(...)` and `^properties(...)`
- a Vite-first compilation path
- editor, lint, type-checking, and formatting support for the authored surface

## Official Authored Files

The official authored file formats are:

- `.litsx`
- `.litsx.jsx`

Plain `tsx/jsx` can still participate as compatibility paths, but they are not
the primary authored product surface.

## First-Wave Tooling

The first public release includes:

- `vscode-litsx`
- `@litsx/typescript-plugin` and `litsx-tsc`
- `@litsx/eslint-plugin`
- `prettier-plugin-litsx`
- `@litsx/vite-plugin`
- `create-litsx-app`
- the LitSX runtime, compiler, and public Babel packages

## Intentional Limits In 0.1.0

Some surfaces stay intentionally narrow in the first release:

- `prettier-plugin-litsx` formats `.litsx` and `.litsx.jsx`, not plain `tsx/jsx`
- the dedicated `LitSX` VS Code language modes do not replace the full built-in
  JavaScript or TypeScript language services
- internal docs tooling packages such as `@litsx/vitepress` stay out of the
  first public publish wave
